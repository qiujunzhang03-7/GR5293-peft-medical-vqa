# Contributing

This is a course project repo. A few conventions to keep git log readable.

- Qiujun Zhang (qz2579)

## Branches

We use feature branches that merge into main via PR:

- main
- data (data pipeline & baseline)
- lora (LoRA experiments)
- qlora (QLoRA experiments)
- dora (DoRA experiments)
- analysis (cross-method comparison)
- demo (Gradio demo)

## Commit messages

Format: [name] area: summary

Areas: data, eval, train, docs, tests, ci, demo, qlora, dora, analysis.

## Pull requests

- Open a PR when something is review-ready.
- Tag a teammate to review before merging.
- Don't force-push to main.
